在日常工作中，我经常遇到这样的场景：拿到一个日志时间戳（如 250610-163134），还需要结合某个以秒为单位的偏移值，还原真实发生的时间。

为此，我编写了一个小脚本 calculate_timestamp.py，使用 Python 的标准库 argparse 和 datetime，帮助我快速完成转换。

📌 问题背景

日志分析或调试某些系统问题时，经常会遇到“时间戳 + 秒数差”这样的数据组合。手工换算效率低且容易出错，于是我决定用脚本来解决。

🛠️ 脚本功能说明

该脚本支持从命令行输入以下两个参数：

  • --seconds / -s: 要减去的秒数（支持浮点数）
  • --timestamp / -t: 原始时间戳，格式为 YYMMDD-HHMMSS
